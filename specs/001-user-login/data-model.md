# Data Model: User Login

## Entities

### User

Managed by Supabase Auth in the `auth.users` table. Not directly queryable from client code.

| Field | Type | Description |
|-------|------|-------------|
| `id` | UUID | Primary key, auto-generated by Supabase |
| `email` | string | Unique email address, verified format |
| `created_at` | timestamp | Account creation timestamp |
| `last_sign_in_at` | timestamp | Last successful sign-in |
| `email_confirmed_at` | timestamp | When email was confirmed (nullable) |

**Relationships**: User has many Sessions.

### Session

A browser session associated with an authenticated user. Managed by Supabase Auth and persisted via cookies through `@supabase/ssr`.

| Field | Type | Description |
|-------|------|-------------|
| `access_token` | JWT | Short-lived access token (default 1 hour) |
| `refresh_token` | JWT | Long-lived refresh token (default 60 days) |
| `expires_at` | timestamp | When the access token expires |
| `user_id` | UUID | References `auth.users.id` |

**Note**: Session data is stored in HTTP-only cookies, not in a custom database table. `@supabase/ssr` handles cookie serialization and token refresh automatically.

### PasswordResetToken

A time-limited token that authorizes a password change. Managed entirely by Supabase Auth — no custom table needed.

| Field | Type | Description |
|-------|------|-------------|
| `token_hash` | string | Hashed token stored in `auth.users` recovery fields |
| `expires_at` | timestamp | Token expiry (default 1 hour) |
| `user_id` | UUID | References `auth.users.id` |

**Note**: Supabase Auth manages recovery tokens internally in `auth.users.recovery_token` and `auth.users.recovery_sent_at`. The flow is: user requests reset → Supabase sends email with link → `auth/callback` route handles token exchange → user redirected to update-password page.

## Validation Rules

| Field | Rule | FR Reference |
|-------|------|--------------|
| Email | Must match `/^[^\s@]+@[^\s@]+\.[^\s@]+$/` | FR-002 |
| Password | Minimum 8 characters, at least 1 uppercase, 1 lowercase, 1 number | FR-003 |
| Login attempts | Max 5 failed attempts per IP per 15 minutes | FR-013 |
| Reset requests | Max 3 password reset requests per email per hour | FR-014 |
| Reset token expiry | Tokens expire 1 hour after issuance | FR-009 |

## State Transitions

```
[Unauthenticated] 
    → signUp(email, password) → [Authenticated]  (FR-001)
    → signIn(email, password) → [Authenticated]  (FR-004)
    → requestReset(email)     → [ResetSent]     (FR-008)

[Authenticated]
    → signOut()               → [Unauthenticated]  (FR-006)
    → access protected route  → [Authenticated]    (FR-007)

[ResetSent]
    → validResetLink(password) → [Authenticated]   (FR-010)
    → expired/invalid link     → [Unauthenticated] (FR-010 edge case)
```
